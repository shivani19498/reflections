Q-1.When would you want to use a remote repository rather than keeping all your work local?

Ans.When I intend to share my work with public.

Q-2.Why might you want to always pull changes manually rather than having Git automatically 
stay up-to-date with your remote repository?

Ans.Sometimes,I may not be working on my computer or I may not have made the changes myself.
In such situations,I will need to pull changes manually.

Describe the differences between forks, clones, and branches. When would you use one 
instead of another?

Forking a repository means copying someone else's repository to you github profile.
Cloning a repository refers to copying a repository to your local machine using the URL.
Branching refers to adding some changes to your own repository and committing them and 
giving them a label.
